Owner wants to iterate quickly. Large combined briefs (two tasks in one) slow down the cycle because the agent reads more, thinks more, writes more.

**Why:** Explicitly stated: "snel en gericht is pre, ik wil snel itereren."

**How to apply:** One brief = one task. When multiple things need to happen sequentially (e.g. spec then implement), run them as separate agents in sequence rather than bundling into one. Keep each brief to the minimum scope needed to unblock the next step.
