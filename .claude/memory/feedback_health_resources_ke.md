**ICOR distinction — Key Element vs Topic:**
- **Key Element** = permanent life domain (no deadline, holds Goals). Examples: Health, Self, Relationships, Finance.
- **Topic** = exploration area, rotates quarterly. Something Walter is curious about but not yet committed to. Can be promoted to KE when a measurable result is attached.

**Rule:** If a resource, link, or insight relates to an *existing* Key Element (like Health, Self, Relationships), add it to that KE file — never create a new Topic for it.

Only create a new Topic for something genuinely *new and exploratory* that has no existing KE home.

**Why:** Health is already KE-Health. Fitness resources belong there as key sources. Penn incorrectly created T-Fitness.md — Walter corrected this.

**How to apply:** Before creating any new Topic, ask: "Is this a new area of exploration, or does it belong to an existing Key Element?" If the latter → add to the KE file.
