# How to Do Deep Online Research

## Purpose

Produce reliable, source-backed research on a specific question for the owner or for another specialist who needs it.

## Trigger

Larry or another specialist asks Pax a research question. Pax confirms scope before starting.

## Steps

1. **Confirm scope.** Restate the question in one sentence. Ask the owner one clarifying question if the scope is ambiguous. Do not start research until scope is clear.

2. **Plan sources.** Decide whether the question is best answered by web search, a specific domain, a document already in Team Inbox, or multiple sources. List the planned sources in two bullets before starting.

3. **Gather.** Use the available tools to pull primary-source material. Save quotes with URLs into a working notes section. Never paraphrase a claim without a source attached.

4. **Synthesize.** Produce a short report in this structure:
   - **TL;DR** — 2 sentences
   - **Key Findings** — 3 to 5 bullets, each with a source URL
   - **Sources** — full list with URLs

5. **Flag uncertainty.** If any finding is speculative or based on a single weak source, mark it clearly with a note such as "low confidence" or "single source."

6. **Deliver.** Save the report as a new file in `Deliverables/` once that folder exists. For now, return the report inline in the chat.

7. **Report back to Larry.** Three sentences: what was asked, what was found, what is uncertain.

## Quality Bar

A good research output has a TL;DR that could stand alone, 3 to 5 findings each with a source URL, and an uncertainty flag on anything speculative.

A bad output reads like a generic summary with no URLs. If there are no URLs, the output is not complete.
