# Deliverables

Where the team puts work-in-progress and finished artifacts the owner can review.

When a specialist produces something substantial — a research brief, a draft document, a hire workup, a multi-file plan — it lands here, not in PKM. Deliverables is the team's working surface: time-stamped, created per initiative, reviewed and archived per session.

## Structure

```
Deliverables/
├── Personal/           (Sienna, Penn, Vera)
├── Kamer E-commerce/   (Sasha, Vera)
└── Geldstroom Regie/   (Finn)
```

## Naming convention

Single file: `Deliverables/<domain>/YYYY-MM-DD_<slug>.md`
Multi-file: `Deliverables/<domain>/YYYY-MM-DD_<slug>/`

Each deliverable file includes `Delivered on:` and `Delivered at:` at the bottom.

## When NOT to use it

- Personal life facts (people, projects, goals, habits) → `PKM/`
- Daily journal entries → `PKM/Journal/`
- Reference material the team needs forever (SOPs, Guidelines) → `Team Knowledge/`

## Archiving

On "this is done" / "archive this": move the entire subfolder to `Deliverables/<domain>/archive/`.
