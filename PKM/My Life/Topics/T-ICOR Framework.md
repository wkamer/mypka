> Lesdata — puur bronmateriaal van Paperless Movement / Tom Solid. Geen persoonlijke interpretatie.
> Persoonlijke implementatie → zie [[KE-ICOR]]

---

## Wat is ICOR?

**ICOR** staat voor **Input → Control → Output → Refine**

| Letter | Betekenis | Kern |
|---|---|---|
| **I** | Input | Alles wat binnenkomt — informatie, ideeën, taken, triggers |
| **C** | Control | Verwerken en organiseren zodat je het kunt terugvinden en gebruiken |
| **O** | Output | Resultaten produceren: acties, besluiten, content, werk |
| **R** | Refine | Terugkijken, bijsturen, het systeem verbeteren |

> "Where information meets action." — Paperless Movement

ICOR werkt in drie lagen: **Concepts** (wat en waarom), **Workflows** (hoe), **Implementation** (welke tools).

## Module 1: Note-Taking

*Cursus: Digital Note-Taking Like a Pro — Paperless Movement*

---

### Layer 1: Concepts

#### C01 — What is a note?

Een note is elke vastgelegde eenheid van informatie of actie. Niet alleen tekst — ook een afbeelding, audioclip, video-opname of schets is een note.

**Twee typen inhoud in elke note:**
- **Information** — feiten, inzichten, observaties, ideeën, data
- **Action** — taken, follow-ups, beslissingen, verplichtingen

**Voorbeelden van notes voor professionals:**
Meeting minutes · Task lists · Client notes · Research notes · Project notes · Ideas & brainstorming · Training notes · Conference notes · Personal reflections

**De Inbox:**
Wanneer je een note vastlegt maar nog niet weet waar hij hoort, gaat hij naar de Inbox. De Inbox is een tijdelijke holding area — geen opslagsysteem. Verwerk hem zodra je kunt. Meerdere Inboxes = chaos. Eén Inbox is ideaal, twee acceptabel.

**Filter bij vastleggen:**
Twee vragen voor elk stuk informatie:
1. Zal ik dit gebruiken?
2. Geeft dit op termijn compound waarde?

Als het antwoord op beide vragen nee is, leg je het niet vast.

---

#### C02 — What is Note-Taking?

Note-taking is het omzetten van bronnen in notes zodat je kunt **herinneren**, **begrijpen** en **actie ondernemen**.

```
SOURCES → NOTES → REMEMBER
                → UNDERSTAND
                → TAKE ACTION
```

**Shallow vs Deep Note-Taking:**

| Type | Doel | Kenmerken | Voorbeelden |
|---|---|---|---|
| **Shallow** | Snel vastleggen, makkelijk terugvinden | Minimale verwerking, hoge snelheid | Telefoonnummer, actie-item, datum, link, beslissing |
| **Deep** | Intellectuele ontwikkeling, verbindingen leggen | Analyse, synthese, eigen woorden | Uitgewerkte notities, essay-achtige verwerking, conceptontwikkeling |

Beide types zijn noodzakelijk. Shallow notes verhinderen mentale overbelasting. Deep notes bouwen kennis op.

---

#### C03 — How to perfectly store a note?

```
CAPTURE → [INTERMEDIATE TOOL] → SAVE → SSOT 1
                                      → SSOT 2
                                      → SSOT 3
                                      → SSOT 4
                                             → RETRIEVE
```

**Drie principes:**

1. **SSOT (Single Source of Truth)** — elke note bestaat op exact één plek. Duplicaten doorbreken de betrouwbaarheid van het systeem.

2. **Context preserved** — een note zonder context verliest zijn waarde. Elke note heeft:
   - *Tracing* — van waar komt het? (bron, datum, situatie)
   - *Integration* — hoe verbindt het met bestaande kennis?

3. **Retrievability** — een note is waardeloos als je hem niet kunt terugvinden. Bouw minimaal twee retrieval-paden: folder + tag, persoon + topic, etc.

**Praktijkvoorbeeld:**
- Zonder context: *"Users want faster onboarding."*
- Met tracing: *"Users want faster onboarding. Source: Customer feedback session, Feb 12, 2026, Enterprise segment."*
- Met integratie: gekoppeld aan Onboarding Improvement project + Q1 tag

Nu vindbaar via zoekterm, projectfolder én prioriteitstag.

---

#### C04 — How to link notes?

Drie methoden om notes te verbinden:

| Methode | Werking |
|---|---|
| **Folders & Tags** | Groeperen op categorie — één note in één folder, tags voor meerdere contexten |
| **Link & Backlink** | Directe verwijzing van note A naar note B; backlink wordt automatisch aangemaakt |
| **AI** | Semantische verbindingen op basis van inhoud — vindt relaties die je zelf niet zag |

Meerdere verbindingsmethoden = meerdere retrieval-paden naar dezelfde note.

---

#### C05 — What is the best note-taking app?

Er bestaat geen universeel beste app. Evalueer op vijf dimensies:

| Dimensie | Opties |
|---|---|
| **How they store information** | Local / Cloud |
| **How you create content** | Visual / Outliner / Long-form text (Markdown, proprietary, word processor) |
| **How they organize notes** | Folders / Tags / Links / Backlinks / AI |
| **Level of connection** | Note-level / Block-level |
| **Target audience** | Developers / Researchers / Students / **Busy Professionals** |

ICOR richt zich op **Busy Professionals** — de combinatie van criteria hangt af van jouw workflow.

```
START
  ├── DESKTOP
  │     ├── Information → INFORMATION SSOT
  │     └── Action → ACTION SSOT
  └── MOBILE

```

**Kernprincipe:** Capture eerst, organiseer later. De Inbox is het vangnet wanneer je niet direct naar de SSOT kunt.

1. Dient dit mijn huidige werk?
2. Dient dit mijn doorlopende verantwoordelijkheden?
3. Dient dit mijn actieve exploratieonderwerpen?

Als geen van de drie: laat het los. FOMO is de stille saboteur van dit workflow — het systeem werkt juist correct als je dingen laat passeren.

## Module 2: PKM Like a Pro

*Cursus: PKM Like a Pro — Paperless Movement*

---

### Layer 1: Concepts

#### C01 — What is PKM and a PKM System?

**PKM** (Personal Knowledge Management) is een proces dat informatie omzet in actie die is afgestemd op jouw doelen — geen app, geen archiefsysteem.

PKM beslaat de volledige Knowledge Management Cycle:

```
DISCOVER → CAPTURE → PROCESS → ACTION → SHARE
```

Note-taking is slechts één stap (Capture). PKM dekt de hele levenscyclus van informatie.

PKM zit in het persoonlijke informatiekwadrant van het ICOR Framework, naast BKM (Business Knowledge Management), PPM en BPM.

**Gouden regel:** Definieer eerst je processen, dan pas tools. Tools zijn vervangbaar; processen niet.

> "We shape our tools, and thereafter our tools shape us." — Marshall McLuhan

---

#### C02 — Why Do You Need a PKM System?

Vier redenen:

1. **Idea Capturing** — ideeën verdwijnen zonder systeem
2. **Leverage & Compound** — kennis groeit in waarde naarmate het verbindt
3. **The Power of Thinking** — een systeem verlengt je denkvermogen
4. **Convert information into action** — de brug van weten naar doen

> "Perhaps millions of people, in the last few thousand years, have had ideas for improving it. All I did was take things a little further than just having the idea." — James Dyson

---

#### C03 — Shallow Thinking vs. Deep Thinking

| Type | Proces | Uitkomst |
|---|---|---|
| **Shallow Thinking** | Capture ↔ Retrieve | Snel vastleggen en terugvinden |
| **Deep Thinking** | Meerdere ideeën verbinden → Understanding → Action | Inzicht en beslissing |

Beide zijn nodig. Shallow houdt de flow gaande. Deep bouwt echte kennis op.

---

#### C04 — What is Context?

Context = Tracing + Integration.

```
SOURCES → [TRACING] → PKM SYSTEM
                          BUCKET 1 ─┐
                          BUCKET 2  │ INTEGRATING
                          BUCKET 3  │
                          BUCKET 4 ─┘
```

- **Tracing** — van waar komt de informatie? (bron, datum, situatie)
- **Integrating** — hoe verbindt het met wat al in het systeem zit?

Zonder context verliest een note zijn waarde zodra je hem opslaat.

---

#### C05 — What is an Inbox?

```
IDEAS → CAPTURE → INBOX → SSOT 1 ─┐
                         → SSOT 2  │→ RETRIEVE
                         → SSOT 3  │
                         → SSOT 4 ─┘
```

De Inbox is de tijdelijke landing zone voor alles wat je vastlegt voordat je weet waar het hoort. Capture eerst, organiseer later. Verwerk de Inbox dagelijks.

---

#### C06 — What is a Bucket?

Een Bucket is een container in je PKM Systeem. Buckets kunnen genest zijn: Bucket A bevat Bucket 1 en Bucket 3. Buckets op hetzelfde niveau staan naast elkaar.

Het PKM Systeem is de verzameling van alle buckets. Elk stuk informatie hoort in precies één bucket (SSOT).

---

#### C07 — Outer World vs. Inner World

| | Outer World | Inner World |
|---|---|---|
| **Wat** | Artikelen, videos, podcasts, rapporten, boeken | Meeting notes, reflecties, strategische inzichten, analyses, ideeën |
| **Gemaakt door** | Anderen | Jijzelf |
| **Noise/Signal** | Veel noise, weinig signal | Weinig noise, veel signal |
| **Uitdaging** | Filteren | Vastleggen & organiseren |

Beide werelden voeden je PKM — met verschillende uitdagingen.

---

#### C08 — MY LIFE

MY LIFE is de masterstructuur van je PKM. Vier buckets:

```
MY LIFE
  ├── CURRENT PROJECTS  (actief werk, start- en einddatum)
  ├── KEY ELEMENTS      (doorlopende levensgebieden, geen deadline)
  ├── TOPICS            (exploratiegebieden, roteert per kwartaal)
  └── HABITS            (terugkerende gedragingen met cadans)
```

- **Projects** worden gearchiveerd zodra ze klaar zijn.
- **Key Elements** zijn permanent zolang het levensgebied relevant is.
- **Topics** roteren — een Topic dat een meetbaar resultaat krijgt, wordt gepromoveerd naar Key Element.
- **Habits** ondersteunen doelen via herhaling.

---

#### C09 — GOALS — The Layer That Turns Structure Into Movement

Goals zijn de operationele laag bovenop MY LIFE. Zonder Goals is de structuur statisch.

**The Anchoring Rule:**
> Every Goal anchors to a Key Element. Never a Project. Never a Topic.

Twee tot drie Goals per kwartaal. Richting van beweging omhoog.

Voorbeelden:
- *Lose 10 kg by Q3* → ankert aan **HEALTH**
- *Grow business 30%* → ankert aan **COMPANY**
- *Be more present* → ankert aan **FAMILY**

---

#### C10 — The Promotion Path

Een Topic dat exploratief begint kan promoveren naar Key Element zodra er een meetbaar resultaat aan wordt verbonden.

```
TOPIC (French)          →  GRADUATION                →  KEY ELEMENT (French)
rotates quarterly          signal: "I want to commit     permanent · multi-year
exploration only           to a measurable result"       holds Goals
                                                              ↓
                                                         GOAL ANCHORED
```

De promotie duurt twee minuten: hernoem het bucket, verplaats naar Key Elements, verankert het Goal.

Het omgekeerde geldt ook: als iets niet meer permanent is, archiveer het en sluit de bijbehorende Goals.

> "The structure isn't recognizing what something is. It's recognizing what something has become." — Paperless Movement

### Layer 2: Workflows

#### W01 — The Goal-Setting Workflow

```
1. ANCHOR          →  2. ANCHOR TO KEY ELEMENT  →  3. DECOMPOSE
   Name the              Permanent ground              Into shapes:
   destination                                         PROJECT (one-time, start/end)
                                                        HABIT (recurring rhythm)
```

**The Constraint:** Een Goal kan alleen worden bereikt via een Project of een Habit. Er is geen derde vorm.

---

#### W02 — The Capturing Beast

De Capturing Beast is het filtermodel dat bepaalt of iets de moeite waard is om vast te leggen.

Vier filtervragen:

| Filter | Vraag |
|---|---|
| **Key Elements** | Does this relate to a Key Element? |
| **Projects** | Does this serve a current Project? |
| **Habits** | Does this support an active Habit? |
| **Topics** | Is this connected to a Topic I'm exploring? |

Als geen van de vier van toepassing is: laat het gaan.

---

#### W03 — Workflow 1: DISCOVER

```
DISCOVER > capture > process > action ········ share

THE CAPTURING BEAST
  ├── CURRENT PROJECTS
  ├── KEY ELEMENTS
  └── TOPICS
```

Discover = actief zoeken naar informatie die relevant is voor jouw buckets. De Capturing Beast filtert wat de moeite waard is om vast te leggen.

---

#### W04 — Workflow 2: CAPTURE

```
discover > CAPTURE > process > action ········ share

IDEAS → Capturing Beast → INBOX → Noise → 🗑️
                                → Signal → PKM SYSTEM (Buckets)
```

Twee inboxes:
- **Outer World inbox** — externe bronnen (artikelen, links, rapporten)
- **Inner World inbox** — eigen ideeën, reflecties, observaties

Snelheid is alles. Als vastleggen meer dan vijf seconden kost, sla je het over onder druk. Capture nu, evalueer later.

> "The palest ink is better than the best memory." — Chinese spreekwoord

---

#### W05 — Workflow 3: PROCESS

```
discover > capture > PROCESS > action ········ share

IDEA → THE THOUGHT LAB → Worth it?
                              YES → ACTION → PKM SYSTEM
                                    Shallow Thinking System
                                    Deep Thinking System
                              NO  → discard
```

The Thought Lab is de evaluatiestap: is deze informatie het waard om te investeren? Ja → verwerk. Nee → gooi weg. Er is geen tussenweg.

---

#### W06 — Workflow 4: ACTION

```
discover > capture > process > ACTION ········ share

PKM SYSTEM
  Shallow Thinking → ACTION → Task Management System ✓
  Deep Thinking    → ACTION → Task Management System ✓
```

Twee paden:
- **Path 1** — je weet wat te doen: schrijf een concrete volgende stap, stuur naar PPM (persoonlijk) of BPM (team)
- **Path 2** — je hebt meer denkwerk nodig: verplaats naar Deep Thinking workspace

Kennis blijft in PKM. Acties verplaatsen naar buiten.

> "Vision without execution is hallucination." — Thomas Edison

---

#### W07 — Workflow 5: SHARE

```
discover > capture > process > action ········> SHARE

PERSONAL WORLD                    BUSINESS WORLD (team/outer)
Deep Thinking System              My Team / Outer
  ↓ Worth it to share?              Project Management System
  YES →                               The Idea Incubator
  NO  → stays private
```

Niet alles hoeft gedeeld te worden — de meeste verwerkte kennis blijft privé. Share alleen wat anderen genuïne waarde geeft.

Delen stress-test je ideeën, onthult blinde vlekken en sluit de cyclus: feedback van anderen creëert nieuwe informatie om te ontdekken.

PKM ontmoet BKM hier: persoonlijke inzichten worden teamdocumentatie, voorstellen of gepubliceerde content.

## Module 3: Task Management Like a Pro

*Cursus: Task Management Like a Pro — Paperless Movement*

> Deze module valt binnen de ACTION-fase van de Knowledge Management Cycle.

---

### Layer 1: Concepts

#### C01 — What is Personal Work?

Deze cursus richt zich op **persoonlijk werk** (werkweek, ma-vr) — niet op het persoonlijke leven. Doel: wekelijkse en dagelijkse planning optimaliseren.

Drie lagen:
1. **Concepts** — principes achter effectief taakbeheer
2. **Workflows** — stap-voor-stap processen
3. **Implementation** — tools kiezen die de processen ondersteunen

---

#### C02 — Shallow Work vs. Deep Work

*(Concept van Cal Newport — "Deep Work: Rules for Focused Success in a Distracted World")*

| | Shallow Work | Deep Work |
|---|---|---|
| **Cognitief** | Niet veeleisend | Volledige focus vereist |
| **Waarde** | Weinig nieuwe waarde | Nieuwe waarde |
| **Repliceerbaarheid** | Makkelijk te repliceren | Moeilijk te repliceren |
| **Tijdshorizon** | Dagelijkse operaties | Mid- tot langetermijn doelen |
| **Voorbeelden** | Emails beantwoorden, admin, meetings, browsen | Bedrijfsstrategie, nieuw ontwerp, vaardigheid leren |

**ICOR aanpak — vier standaard tijdblokken per dag:**
1. **Deep Work** — 2-3 uur ononderbroken, 's ochtends als eerste
2. **Morning Routine** — dag plannen, inboxes checken
3. **Afternoon Routine** — inboxbeheer, delegeren
4. **End-of-Day Routine** — journaling, morgen plannen

> "Two to three hours of uninterrupted, focused deep work can produce higher quality output and greater progress than an entire day filled with distracted, shallow work." — Cal Newport

---

#### C03 — Tasks vs Speedies vs Reminders

Classificatie = de action-kant van de Capturing Beast.

| Type | Duur | Werking |
|---|---|---|
| **Task** | 15 min – 3 uur | Gaat in Deep Work blok |
| **Speedy** | < 15 min | Wordt gebatched |
| **Reminder** | n.v.t. | Tijdgebonden alert — instellen en vergeten |

Zonder classificatie weet je niet wat je plant of hoe lang het duurt. Alle planning-workflows hangen hiervan af.

---

#### C04 — How to Tackle Tasks

Een goed geformuleerde taak heeft vier elementen:

```
TASK = ACTION VERB + OUTCOME + SCHEDULE + TIME SPAN
```

Voorbeeld: *"Write essay tomorrow from 9am to 10am."*
- Action verb: Write
- Outcome: essay
- Schedule: tomorrow
- Time span: from 9am to 10am

---

#### C05 — Tasks on Non-task Management Tools

Taken ontstaan overal — maar horen op één plek: het **Task Management SSOT**.

Bronnen die taken genereren:
- Communication tool (Slack, Teams)
- Email tool
- Shallow Thinking tool (PKM)
- Speedies tool
- Planner tool

Alle items worden doorgestuurd naar de Task Management SSOT. Geen verspreide lijsten.

---

#### C06 — Weekly Goals and Highlight of the Day

- **Weekly Goals** — 2-3 doelen voor de week, verankerd aan Key Elements
- **Highlight of the Day** — de ene taak die vandaag het meest telt; gemarkeerd met ster (⭐)

De Highlight bepaalt de volgorde van je dag. Als er maar één taak gedaan wordt, is het de Highlight.

---

#### C07 — Planning Doesn't Mean Doing

Planning is voorbereiding, geen garantie. Drie stappen:

```
1. List all tasks
→ 2. Pick Highlight of the Day (⭐)
→ 3. Balance against unexpected tasks
```

De weegschaal: geplande taken vs. onverwachte taken. Flexibiliteit is ingebouwd — niet alles gaat zoals gepland.

---

#### C08 — Routines

Routines zijn vaste gedragingen die op de automatische piloot draaien en Goals ondersteunen.

```
ROUTINES → GOAL
```

> "You'll never change your life until you change something you do daily. The secret of your success is found in your daily routine." — John C. Maxwell

Drie dagelijkse routines: Morning · Afternoon · End-of-Day

---

#### C09 — Balance

Een gebalanceerde dag heeft vier soorten blokken:

```
HIGH LEVEL ENERGY TASK (Deep Work)
UNPLANNED EMPTY SPACE
LOW LEVEL ENERGY TASK (Shallow Work)
UNPLANNED EMPTY SPACE
```

Herhaal dit patroon door de week. Lege ruimte is geen verspilling — het is buffer voor het onverwachte.

> "Balance is not something you find, it's something you create." — Jana Kingsford

---

#### C10 — Batching

Groepeer gelijksoortige Shallow Work taken in één blok. Voorbeelden:

- Speedies
- Email
- Administratieve taken
- Teamcommunicatie
- Shallow Thinking Inbox verwerken

Batching vermindert context-switching en beschermt je Deep Work tijd.

---

#### C11 — Time Blocking

Wijs specifieke tijdblokken toe aan specifieke taaktypen. Stel een tijdslimiet in.

> Parkinson's Law: "Work will take as much time as is available for it to be completed." — Cyril Northcote Parkinson

Voorbeelddag (slide):
Morning Routine → Deep Work blok → Lunch → Afternoon Routine → Manage Email → Wrap Up

---

#### C12 — Sequentiality

Doe één taak tegelijk.

```
TASK → TASK → TASK → TASK
```

Drie uitkomsten: **Focus** · **Accomplishment** · **Prioritize**

Multitasking verbreekt aandacht. Sequentieel werken bouwt momentum op.

### Layer 1: Concepts

#### C01 — What is a Project Management System?

Een Project Management System heeft vijf componenten:

1. **Organize and plan team tasks** — gestructureerd overzicht van teamwerk
2. **Set priorities** — duidelijke keuzes over wat eerst gaat
3. **Goal alignment** — iedereen werkt aan hetzelfde doel
4. **Convert ideas into tasks** — ideeën worden uitvoerbare acties
5. **Seamless team communication** — communicatie zit aan het werk vast

Schaalbaar voor elk teamformaat: solo + contractors, small team (2-8), large team (8-25), large organization (25+). De concepts blijven gelijk; de implementatie schaalt mee.

---

#### C02 — What Are the Output Elements?

```
OUTPUT ELEMENTS
  ├── ACTIONABLE
  │     └── TASKS ✓
  └── NON-ACTIONABLE
        ├── GOALS 🎯
        ├── PROJECTS 📁
        ├── WORKSTREAMS 🔄
        └── OPERATIONS 🔧
```

- **Tasks** — uitvoerbare acties (actie-verb + outcome + schedule + tijdspan)
- **Goals** — meetbare resultaten, verankerd aan Key Elements
- **Projects** — eenmalige inspanning met start- en einddatum
- **Workstreams** — terugkerende multi-stap processen
- **Operations** — doorlopende dagelijkse activiteiten zonder deadline

---

#### C03 — What is the OUTPUT Formula?

Twee richtingen om het systeem te bouwen:

**Top-Down** (strategisch → operationeel):
```
GOALS → PROJECTS / WORKSTREAMS / OPERATIONS → TASKS
```

**Bottom-Up** (operationeel → strategisch):
```
TASKS → PROJECTS / WORKSTREAMS / OPERATIONS → GOALS
```

Beide zijn geldig. Top-Down start vanuit richting. Bottom-Up start vanuit wat er al loopt en werkt omhoog naar alignment.

---

#### C04 — What is PEA?

PEA is het dagelijkse uitvoeringsframework:

```
PEA
  ├── PLAN    — WHEN am I going to do this?
  ├── EXECUTE — WHAT do I do now?
  └── ALIGN   — Am I doing what I SHOULD be doing?
```

> "If you have more than three priorities, you don't have any." — Jim Collins

### Layer 2: Workflows

#### W01 — The Execution Beast

De centrale workflow die kwartaaldoelen vertaalt naar dagelijkse uitvoering.

```
CREATE Output Elements
  (Goals · Projects · Workstreams · Operations · Tasks)
        ↓
       PEA
        ↓
  PLAN ─────────────────────────────────────────────────
  Project Management System
    → Weekly Planning
          → This week
          → Next week

  EXECUTE ──────────────────────────────────────────────
  Task Management System
    → Daily Planning
          → EXECUTE
                → Energy Level
                → Urgent
                → Speedy
                → Unexpected Event

  ALIGN ────────────────────────────────────────────────
  Am I doing what I SHOULD be doing?
```

---

#### W02 — The Idea Incubator

Workflow voor het omzetten van ideeën naar uitvoering.

```
THE IDEA INCUBATOR
  → CAPTURE  (SSOT — één plek voor alle ideeën)
  → EVALUATE (To be planned / Someday / Rejected)
  → PLAN
  → EXECUTE  (via Execution Beast)
```

Ideeën worden niet direct taken. Ze gaan eerst door evaluatie. Alleen "To be planned" ideeën bereiken de Execution Beast.

---

#### W03 — The Team Communication System

```
TEAM COMMUNICATION MANAGEMENT SYSTEM
  ├── ASYNCHRONOUS         (default communicatiemodus)
  ├── WEEKLY MEETINGS      (Weekly Goals + projectvoortgang)
  ├── "TO REVIEW" STATUS   (expliciete reviewfase voor deliverables)
  └── SSOT                 (alle info op één plek)
        ↓
  BKM (Business Knowledge Management)
        ├── TRUST
        ├── EFFICIENCY
        ├── FOCUS
        └── EXECUTION
```

Asynchronous is de standaard — niet meetings. Meetings alleen voor wat echt synchronisatie vereist. Communicatie zit vast aan het werk (SSOT), niet verspreid over Slack-threads.
