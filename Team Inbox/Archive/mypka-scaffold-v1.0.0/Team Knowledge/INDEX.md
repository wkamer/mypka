# Team Knowledge - Master Hub

This is the operations side of the vault. It holds the team's procedures, orchestrations, reference material, and session history. The user's personal knowledge lives in [[PKM/INDEX]].

## Sections

- **[[Team Knowledge/SOPs/INDEX|SOPs]]** - atomic step-by-step procedures. One job per file. Filenames: `SOP-NNN-<title>.md`.
- **[[Team Knowledge/Workstreams/INDEX|Workstreams]]** - recurring multi-agent orchestrations. Reference SOPs and Guidelines via `[[wikilinks]]`. Filenames: `WS-NNN-<title>.md`.
- **[[Team Knowledge/Guidelines/INDEX|Guidelines]]** - static reference info (naming, tone, defaults). SOPs and Workstreams point at these. Filenames: `GL-NNN-<title>.md`.
- **session-logs/** - append-only record of every working session, written by Larry. Path: `session-logs/YYYY/MM/YYYY-MM-DD-<slug>.md`.

## Taxonomy in plain English

- An **SOP** answers "how do I do X?" in clear steps.
- A **Workstream** answers "how do we do X every day or every week, with multiple specialists?" It choreographs SOPs.
- A **Guideline** answers "what is the rule for X?" Static reference. Never a procedure.

When in doubt, write a Guideline first if the rule is static. Write an SOP if the steps repeat. Write a Workstream when more than one specialist is involved.

## SSOT applies here too

If naming rules belong in [[GL-001-file-naming-conventions]], do not restate them inside an SOP or Workstream. Link to the Guideline instead.
